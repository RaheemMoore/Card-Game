# Fantasy Character Naming Bible

**Version:** 1.0  
**Project:** Card Game  
**Purpose:** Canonical naming reference for generated character cards  
**Primary users:** Claude Code, Claude API prompt generation, lore designers  
**Scope:** Character names and epithets for all current archetypes  
**Current archetypes covered:** Barbarian, Monk, Beastmaster, Druid, Necromancer, Vampire, Lycanthrope, Mech Pilot, Android, Seraph, Human

---

# 1. Why This Resource Exists

Character names must feel like they belong to the person being created.

The current generation system selects a random cultural naming style from a shared global list, then asks Claude to produce a fantasy name in that style. The style is not reliably connected to:

- archetype
- lineage
- setting
- demeanor
- character history
- technological or magical culture
- visual identity

This can create characters whose names feel unrelated to their appearance or lore. A druid raised in an ancient forest, a synthetic android, and a gothic vampire may receive names built from the same naming logic. Small example lists also encourage repeated syllables, familiar fantasy defaults, and recycled names.

This Bible gives Claude enough structure and examples to create names that feel culturally grounded, archetype-aware, varied, and memorable without copying real people or relying on stereotypes.

---

# 2. Core Naming Principle

A character's name should answer at least three questions:

1. **Where could this person or being have come from?**
2. **What kind of culture, community, maker, order, bloodline, or environment named them?**
3. **Why does the name fit this specific character rather than any random fantasy hero?**

Archetype influences a name, but archetype must never be the only input.

A Barbarian does not automatically need a Norse-style name.  
A Monk does not automatically need an East Asian-style name.  
A Druid does not automatically need a Celtic-style name.  
A Necromancer does not automatically need a Latin name.  
A Human does not automatically need an ordinary modern American name.

Use the full character context.

---

# 3. Naming Input Priority

Build the name from the following inputs in order:

1. **Locked existing name**  
   Never rename an evolving character.

2. **Lineage or constructed origin**  
   Human culture, synthetic manufacturing origin, divine order, vampiric house, wolf pack, monastery, clan, grove, or machine series.

3. **Setting and implied culture**  
   Desert empire, mountain hold, drowned city, neon fortress, moonlit forest, ruined monastery, and so on.

4. **Archetype naming tendencies**  
   Use the relevant archetype chapter in this document.

5. **Character identity**  
   Gender presentation, age, body type, demeanor, rank, and distinctive history.

6. **Modifiers and Whisper results**  
   Signature detail, class trait, element, lighting, and other visible narrative clues.

7. **Phonetic variety and collision checks**  
   Avoid names that resemble recent cards or overused fantasy defaults.

---

# 4. Cultural Inspiration Rules

Real cultures may inspire sound, rhythm, structure, and naming customs, but they must be used carefully.

## Do

- Describe names as culturally inspired, not historically authentic.
- Use several cultural families across each archetype.
- Match inspiration to the character's setting and lineage.
- Research a culture before adding new canonical rules.
- Transform inspirations into original fantasy forms.
- Preserve diacritics only when the system reliably supports them.
- Let characters of every gender appear in every naming tradition.
- Use clan names, house names, order names, maker marks, and earned names where appropriate.

## Do Not

- Assign one real culture permanently to one archetype.
- Mix random syllables from unrelated languages.
- Use sacred titles, deities, ethnic groups, or famous historical names casually.
- Treat one culture as savage, evil, primitive, mystical, or technologically superior.
- Add apostrophes merely to make a name look fantastical.
- Use real-world surnames as shorthand for ethnicity.
- Copy recognizable names from existing fantasy franchises.
- Assume a character's physical appearance determines their culture.

# 5. Name Construction System

Each generated character should use one of these structures.

## Structure A — Personal Name

`Given name`

Best for solitary figures, mysterious characters, and beings who discarded an earlier identity.

Examples:

- Veyra
- Omari
- Tamsin
- Davor
- Soryn

## Structure B — Personal and Family Name

`Given name + family name`

Best for Humans, Vampires, Mech Pilots, and characters from established nations, cities, or houses.

Examples:

- Sabine Valcor
- Idris Marr
- Nia Calder
- Tomas Vey
- Soraya Dastan

## Structure C — Personal and Clan Name

`Given name + clan or pack name`

Best for Barbarians, Beastmasters, Lycanthropes, and kinship-based societies.

Examples:

- Brenna Ash-Tusk
- Kato of the Reed Fang
- Amara Moontrail
- Dren Stonehide

## Structure D — Order or Place Name

`Given name + of/from the order or place`

Best for Monks, Druids, Necromancers, Seraphs, exiles, and institutional cultures.

Examples:

- Sajan of the Quiet Step
- Elowen of Briarwake
- Merek of the Ninth Crypt
- Ilyra of the Dawn Choir

## Structure E — Earned Name

`Given name + earned byname`

Best for cultures where deeds replace surnames and for Forged or Ascendant reputations.

Examples:

- Osei Stone-Sleeper
- Yara Storm-Bitten
- Veska Three-Scars
- Torren Wolf-Friend

Do not duplicate the epithet. If the card title is `Yara, the Storm-Bitten`, the base card name should normally remain `Yara`.

## Structure F — House or Bloodline Name

`Given name + house name`

Best for Vampires, aristocratic Humans, magical dynasties, and inherited orders.

Examples:

- Lucienne Morcant
- Cassian Veyr
- Zahra Noctevane
- Dorian Vale

## Structure G — Manufactured Designation

`Series, maker, function, or unit designation`

Best for Androids and created beings.

Examples:

- IONA-7
- Axiom R4
- Kestrel-12
- NERA // 03
- Vesper Unit Nine

A designation may later become a chosen personal name. That change should be reflected in lore, not happen randomly.

## Structure H — Celestial or Liturgical Name

`Personal name + choir, office, vow, or celestial station`

Best for Seraphs and divine orders.

Examples:

- Aurel of the Sixth Radiance
- Samara Dawn-Vigil
- Orien of the Brazen Choir
- Caelis Mercy-Bound

Avoid directly reusing famous angel names unless explicitly approved.

# 6. General Quality Rules

A strong card name should:

- be pronounceable after one or two readings
- contain one to four spoken beats in most cases
- look distinct from nearby generated cards
- fit the character's lore and visual design
- avoid unearned grandeur at Foundation rank
- remain usable when the character reaches Ascendant
- support a strong epithet without becoming unwieldy
- avoid accidental comedy unless humor is intentional
- avoid direct copies of famous fictional characters
- avoid repeated beginnings such as Kael-, Val-, Aer-, Nyx-, Thal-, or Xy-
- avoid excessive X, Z, apostrophes, and doubled vowels
- not simply describe the archetype, such as “Wolfgar” for every Lycanthrope

## Common Overused Fantasy Patterns

Use sparingly or avoid:

- Kael / Kaelen / Kaelor
- Draven
- Raven
- Shadow
- Night
- Blood
- Grim
- Thorne
- Ash
- Vex
- Nyx
- Xander
- Ael-
- -ion
- -ius
- -ara
- random apostrophes
- “the Destroyer” without a specific deed

These are not forbidden individually. They become a problem through repetition.

# 9. Epithet Rules

Epithets should come from something specific.

Good sources:

- a deed
- a failure
- a place
- a visible trait
- a broken vow
- a relationship
- a legendary battle
- an element used in a distinctive way
- an unusual survival event
- a title granted by a community

## Strong Epithet Examples

- the Gate That Held
- Keeper of the Last Grove
- Who Walked Through Winter
- the Chain-Breaker
- Voice of the Ninth Crypt
- Bearer of Six Wounds
- the Unburned
- of the Hollow Crown
- the Quiet Thunder
- Last of the Reed Fang

## Weak Epithet Examples

- the Powerful
- the Destroyer
- the Dark One
- the Strong
- the Mysterious
- the Legendary
- the Fire Warrior
- the Wolf Man

## Rank Guidance

### Foundation

Names should feel personal, local, or incomplete.

Examples:

- Nessa Reed
- Joren
- Mira, Unit MR-A9

### Forged

The character may gain a recognized byname, office, call sign, or clan standing.

Examples:

- Nessa Reed-Singer
- Joren Scar-Muzzle
- Mira of the Free Circuit

### Ascendant

The title may become legendary, but it must still emerge from established history.

Examples:

- Nessa, Keeper of the Last Green
- Joren, the Howl That Broke Winter
- Mira, First Voice of the Free Circuit

Do not replace the character's base name during evolution unless the lore explicitly depicts a chosen-name transformation.

# 10. Repetition Prevention

The generator must compare a proposed name against existing and recently generated cards.

Reject or regenerate when:

- the exact name already exists
- the normalized name already exists
- the first name was used too recently
- the surname, house, clan, pack, order, or designation is overrepresented
- the first three letters match too many recent names
- the same ending appears too often
- the phonetic shape is too similar to another recent name
- the proposed epithet repeats a recent key noun
- the name strongly resembles a famous fictional character

## Suggested Comparison Windows

At minimum, compare against:

- all permanent card names
- all names generated in the current batch
- the most recent 100 generated characters
- all approved house, clan, order, pack, and manufacturer names

## Useful Normalization

Before comparing:

- convert to lowercase
- remove spaces
- remove punctuation
- remove apostrophes
- remove hyphens
- optionally remove diacritics for collision detection only

Example:

`Kael-Ren`, `Kael Ren`, and `Kael’ren` should be treated as near-collisions.

# 11. Prompt Construction Guidance

Do not ask Claude only:

> Generate a fantasy name using a random cultural style.

Instead provide structured context.

## Recommended Naming Prompt Inputs

- archetype
- lineage
- setting
- demeanor
- physique or age when narratively useful
- signature detail
- class trait
- rank
- whether the character belongs to a clan, house, order, pack, maker, squadron, or grove
- recent names to avoid
- overused phonetic fragments to avoid
- preferred name structure
- whether an epithet is needed

## Recommended Prompt Pattern

> Create one original name for this character. The name must feel native to the character's lineage, setting, archetype culture, and personal history. Use the Fantasy Character Naming Bible as the canonical naming guide. Do not choose a cultural style randomly. Select or construct the naming tradition from the full character context. Avoid stereotypes, famous fictional names, apostrophe-heavy fantasy spelling, and recently used names. Return a pronounceable base name suitable for every rank. Generate an epithet only when the lore provides a specific reason for one.

  ]

# 13. Generation Validation Checklist

Before accepting a generated name, verify:

- Does it fit the archetype without becoming a cliché?
- Does it fit the lineage and setting?
- Does it reflect who named this character?
- Is it pronounceable?
- Is it meaningfully different from recent names?
- Does it avoid a famous franchise resemblance?
- Does it avoid cultural caricature?
- Does the title fit the character's current rank?
- Can the same base name survive future evolution?
- Is the epithet earned by actual lore?
- Does the visual identity support the name?
- Would this name still make sense without seeing the archetype label?

If three or more answers are weak, regenerate the name.

# 14. Implementation Guidance for Claude Code

This Bible should be used as a reference resource by the character-generation pipeline. It should not become one enormous hardcoded prompt copied verbatim into every request.

Recommended implementation:

1. Read the selected archetype chapter.
2. Combine it with lineage, setting, demeanor, and character history.
3. Choose a suitable name structure.
4. Select a culturally coherent sound direction.
5. Generate several internal candidates.
6. Score candidates for coherence, originality, pronunciation, and collision risk.
7. Reject repetitive or franchise-like names.
8. Return the best valid candidate.
9. Store enough naming metadata to explain or reproduce the decision.
10. Preserve the base name through Forged and Ascendant evolution.

Do not hardcode one naming culture per archetype.

---

# 15. Naming Candidate Scoring

A practical internal score can use:

- Character coherence: 0–30
- Cultural and setting coherence: 0–20
- Archetype fit: 0–15
- Originality: 0–15
- Pronounceability: 0–10
- Evolution durability: 0–5
- Collision safety: 0–5

Reject any candidate below 75/100.

Also reject regardless of score when:

- it duplicates an existing name
- it strongly resembles a famous fictional name
- it contains unsupported cultural or sacred terminology
- it contradicts the character's established origin
- it relies entirely on an archetype cliché

- clans
- packs
- houses
- monasteries
- druidic groves
- necromantic academies
- seraphic choirs
- mech squadrons
- android manufacturers
- cities
- nations
- regions

Repeated use of one of these names should indicate a real shared history, not accidental generator repetition.

Examples:

- House Morcant
- Reed Fang Clan
- Quiet Step Monastery
- Elder Grove
- Ninth Crypt
- Dawn Choir
- Cobalt Squadron
- Axiom Foundry

Do not create a new canonical faction merely to generate one character name unless the workflow is allowed to create new lore. When new lore is created and approved, synchronize it with the appropriate project documentation.

# 17. Final Principle

A good fantasy name is not a decorated random word.

It is a compressed piece of worldbuilding.

The name should make the character feel as though they existed before the generation prompt and will continue to exist after the card is created.
